Authentication

Include your API Key in the request header as X-API-Key or as a query parameter ?api_key=YOUR_API_KEY.

Example (using `X-API-Key` header is recommended for security):

curl -H "X-API-Key: YOUR_API_KEY" \
     -X POST \
     -d '{"action": "balance"}' \
     https://yourdomain.com/api/v1

API Actions & Endpoints (Simulated)

1. Get User Balance

Method: POST

Endpoint: /api/v1

2. Place New Order

Method: POST

Endpoint: /api/v1

Body: { "action": "add", "service": "SERVICE_ID", "link": "LINK", "quantity": 1000 }

Response: { "status": "success", "order": "ORDER_ID" }

3. Check Order Status

Method: POST

Endpoint: /api/v1

Body: { "action": "status", "order": "ORDER_ID" }

Response: { "status": "success", "order_id": "ORDER_ID", "api_order_id": "API_ORDER_ID", "current_status": "Processing", "remains": 500 }

Admin Control Panel

This section is for administrators only and would contain features like:

  • User Management (view, edit, suspend users)
  • Service Management (add, edit, delete services/categories)
  • Order Management (change order status, refund)
  • Payment Verification for Manual Deposits
  • Ticket Management (view all tickets, reply to users)
  • API Provider Configuration
  • System Settings & Statistics

For the scope of this request, the focus is on the User Dashboard. A full Admin UI would be developed separately with dedicated authentication and authorization layers.
